Class, Object & OOP

Class

A user-defined data type that acts as a blueprint for objects.

Object

An instance of a class.

Object-Oriented Programming (OOP)

A programming paradigm that organizes software around objects, which combine data and methods, to improve reusability, security, and maintainability.

Benefits of OOPs

1. Code reusability: Because of inheritance, existing code can be reused.
2. Security: Encapsulation protects data from direct access.
3. Easy maintenance: Programs are divided into classes, so finding and fixing errors becomes easier.
4. Modularity: Each class works like a separate module.
5. Flexibility: Polymorphism allows one interface with many implementations.
6. Real-world modeling: OOP matches real-life objects, so programs become easier to understand.

Applications of OOP

  • Software development
  • GUI applications
  • Web applications
  • Mobile applications
  • Banking systems
  • Real-time systems
  • Simulation and gaming
  • Student management systems
  • E-commerce systems

The 4 Pillars of OOP

Super-easy memory:

Abstraction → hide implementation
Encapsulation → hide data
Inheritance → reuse parent features
Polymorphism → one method, many forms

1. Abstraction

The process of hiding implementation details and showing only essential features of an object. 

abstract class Shape {
    abstract void draw();
}

2. Encapsulation

The process of binding data and methods into a single unit and protecting data from unauthorized access.

class Account {
    private int balance = 1000;
    void showBalance() {
        System.out.println(balance);
    }
}

3. Inheritance

The mechanism by which one class acquires the properties and behaviors of another class.

class Animal {
    void eat() { System.out.println("Eating"); }
}
class Dog extends Animal {
    void bark() { System.out.println("Barking"); }
}

4. Polymorphism

The ability of a method/object to take many forms.

  • Overloading: Compile-time ; Same name, different parameters.
  • Overriding: Run-time ; Child class changes parent method.

Java Execution & Architecture

Java is a high-level, object-oriented, secure, portable, and platform-independent programming language.

Platform Independence & Portability

Java compiler generates bytecode, which can run on any system having JVM. Programs can be easily transferred.

Bytecode & Security

Bytecode is the intermediate `.class` file. Java is secure because it avoids unsafe features and runs inside the JVM.

JVM vs JIT

JVM (Java Virtual Machine)

  • Whole virtual machine
  • Provides runtime environment
  • Loads, manages, and executes bytecode

JIT (Just in Time) Compiler

  • Part of JVM
  • Converts bytecode to machine code for speed during execution
One-line memory trick: JVM runs Java, JIT speeds Java.

Java Syntax & Basics

Standard Program Structure

class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}

public: anywhere | static: no object needed | void: returns nothing | main: starting point | String[] args: command-line args.

Data Types

Primitive: int, float. Non-Primitive: String, Arrays.

Variables

Local: in method. Instance: in class. Static: shared.

Type Conversion vs Casting

Conversion: Auto, small to large (Safe).
Casting: Manual (int)x, large to small (May lose data).

Control Flow Statements

Conditional Statements

if / if-else / nested if

Executes code block based on true/false conditions.

if (condition) {
    // true block
} else {
    // false block
}

switch statement

Multi-way selection based on an expression's value. Note: break stops execution; without it, next cases run.

switch(val) {
    case 1: // code
        break;
    default: // code
}

Loops & Jump Statements

Loops (for, while, do-while)

  • while: checks condition first.
  • do-while: runs once first, checks later.
  • for: repeats for a fixed number of times. for(init; cond; update)

Jump Statements (break vs continue)

  • break: stops the loop completely or terminates a switch.
  • continue: skips only the current iteration and moves to the next.
Quick Diff: if is for ranges/conditions, switch is for exact values.

Classes, Objects & Constructors

Class & Object Basics

A class groups data members and member functions. An object is created using: ClassName obj = new ClassName();. Members are accessed using the dot (.) operator.

Constructors

A special member function automatically invoked when an object is created to initialize it. Same name as class, no return type.

Types of Constructors

  • Default: Takes no arguments.
  • Parameterized: Takes arguments to initialize variables.

Constructor Overloading

Having more than one constructor with different parameter lists in the same class.

this Keyword

Refers to the current object of the class. Used to distinguish instance variables from local parameters with the same name.

Student(int rollNo) {
    this.rollNo = rollNo; // current object's var = parameter
}

Pass by Value vs Reference

Best exam line: Java supports pass by value only. For primitive data types, a copy of the value is passed. For objects, a copy of the reference is passed.

Methods & Access Modifiers

Methods

A block of code that performs a specific task. Used to avoid repeating code. Methods can have no return value (void), have a return value (e.g., int), and can take parameters.

Access Modifiers

private: Accessible only inside the same class (Most restricted).
default: Accessible within the same package (no keyword needed).
protected: Accessible within same package AND in subclasses.
public: Accessible from everywhere.

static vs final

static Keyword

Means the member belongs to the class, not objects.

  • Variable: Shared by all objects (e.g., college name).
  • Method: Can be called without creating an object. Only accesses static members.

final Keyword

Means something cannot be changed.

  • Variable: Becomes a constant.
  • Method: Cannot be overridden by child class.
  • Class: Cannot be inherited.
Memory trick: static = shared | final = fixed.
